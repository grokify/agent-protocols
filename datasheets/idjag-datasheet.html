ID-JAG Protocol

Identity Assertion JWT Authorization Grant for Agent Authentication
draft-ietf-oauth-identity-assertion-authz-grant
EXPERIMENTAL

⚠ The Problem

When AI agents act on behalf of users, existing OAuth grants (client credentials, JWT bearer) don't capture which agent acted for which user. Without this, audit logs can't distinguish user actions from agent actions, delegation chains are invisible, and accountability is lost.

✓ The Solution

ID-JAG enables secure agent authentication through JWT assertions with delegation chains. The act claim creates an auditable trail showing exactly which agent acted on behalf of which user, integrating seamlessly with existing OAuth 2.0 infrastructure.

Primary Use Cases

🗓 Calendar Management

Agent schedules meetings on behalf of user with full audit trail

📧 Email Processing

Agent reads and responds to emails with delegated access

🔗 Multi-Agent Chains

Nested act claims track delegation: User → Agent₁ → Agent₂

Protocol Flow (Delegation Mode)

Protocol Parties

👤

Human

Authenticates to IdP; delegates to agent

🏛

IdP / Issuer

Creates and signs ID-JAG assertion

🤖

Agent

Requests assertion; identified in act claim

🔐

Auth Server

Validates IdP signature; issues access tokens

📡

Resource

Protected API; accepts tokens

Quick Start Implementation (Go)

// 1. Request ID-JAG from IdP (token-exchange, requested_token_type=id-jag) idpClient := idjag.NewIDJAGClient("https://idp.example.com/token") idjagResp, _ := idpClient.RequestIDJAG(ctx, &idjag.IDJAGRequest{ SubjectToken: userIDToken, Audience: "https://api.example.com", }) // 2. Exchange ID-JAG at Resource AS (jwt-bearer) jwtClient := idjag.NewJWTBearerClient("https://api.example.com/token") token, _ := jwtClient.Exchange(ctx, idjagResp.AccessToken, "read")

⚡ Performance

  • JWT ~500 bytes typical
  • Validation ~1ms with cached JWKS
  • Cache JWKS with 5-min TTL

🔄 Key Management

  • Rotate signing keys quarterly
